# UAE Pilot Onboarding and Support Model

## Pilot profile

- Cohort size: 5-10 design-partner clients
- ICP: mid-market HR/People Ops (UAE)
- Primary success metric: data accuracy and decision trust

## Onboarding pack

1. **Data template bundle**
   - Employee upload template (required columns + examples)
   - Benchmark upload template (UAE + AED guidance)
   - Validation checklist before upload
2. **Workspace setup checklist**
   - Admin user + workspace settings completed
   - Invite HR/People Ops stakeholders
   - Review cycle and salary view confirmed
3. **Trust setup checklist**
   - Confirm source labels visible in benchmark workflow
   - Confirm demo-vs-real data status is understood
   - Confirm data freshness timestamp is visible

## Support model (pilot)

- SLA targets
  - P1 (accuracy/trust blocker): 1 business hour first response
  - P2 (workflow degradation): 4 business hours first response
  - P3 (non-blocking issue): next business day
- Escalation path
  - CSM triage -> Product owner -> Engineering on-call
- Communication cadence
  - Weekly pilot check-in (30 min)
  - Shared issue tracker with status and ETA

## Weekly governance scorecard

- Inputs reviewed each week:
  - Accuracy incidents (count + severity)
  - Time-to-first-insight (days from kickoff)
  - Workflow completion rate (onboarding, invite, upload, review, report)
  - Stakeholder confidence (1-5 score from pilot sponsor)
- Governance decisions:
  - Continue onboarding next client
  - Pause onboarding until blocker closure
  - Expand or reduce pilot scope
